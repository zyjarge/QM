# 06 · 安全与权限设计

> 文档版本：v1.0 | 日期：2026-07-17

---

## 1. 认证（AuthN）

```
Web 端:  飞书扫码/免登 → OAuth code → QM 换 token(JWT, 2h) + refresh(7d)
IM H5:   飞书工作台免登(自动带 code) → 同上
OpenAPI: App Key + HMAC 签名(供 CI/Git 等系统调用)
```

- JWT 存 Redis 白名单，支持强制下线
- P1 企微接入后：登录页双入口，后端按 provider 分流

## 2. 授权（AuthZ）：RBAC + 数据范围

### 2.1 角色定义

| 角色 | 权限要点 |
|------|---------|
| ADMIN | 全功能 + 模板配置 + 用户管理 + 手动重建群 |
| PM | 需求全操作 + 评审发起 + 归档 + 报表 |
| BIZ_OWNER | 基线签认 + 变更审批 + 评审投票 |
| DEV_LEAD | 技术评审 + 状态流转(dev 段) + 任务关联 |
| QA | 验收标准评审 + 测试关联 |
| REQUESTER | 提交需求 + 验收确认（仅自己提出的） |
| VIEWER | 只读（按数据范围） |

### 2.2 数据范围（行级）

| 规则 | 说明 |
|------|------|
| 干系人可见 | requirement_stakeholders 中有记录即可读 |
| 产品线隔离 | 用户绑定产品线，默认只见本线 |
| 保密需求 | is_confidential=true：仅干系人 + ADMIN 可见，不建群，搜索结果排除 |
| 跨线只读 | 可申请 VIEWER 跨线只读（管理员批） |

### 2.3 字段级

- 成本/工期评估字段：仅 PM + DEV_LEAD 可编辑，干系人只读
- 变更影响评估：仅发起人 + PM 可编辑

## 3. 审计（合规级）

| 项 | 设计 |
|----|------|
| 记录范围 | 全部写操作 + 全部 IM 卡片交互 + 登录 |
| 字段 | actor/action/target/detail(before→after)/ip/ua/时间戳 |
| 防篡改 | 只增不改；每日 hash 链（前一条 hash 入下一条，P2） |
| 留存 | ≥ 3 年，独立于业务库备份 |
| 基线证据 | 签认时 snapshot 全量落 baselines 表 + MinIO 存 PDF 导出（P1） |

## 4. 应用安全清单

| 层 | 措施 |
|----|------|
| 传输 | 全链路 HTTPS（TLS 1.2+），内网也不例外 |
| 注入 | MyBatis 参数化；jsonb 字段禁拼 SQL |
| XSS | 前端统一转义；富文本服务端白名单过滤（TipTap 节点 schema 限制） |
| CSRF | JWT header 认证（非 cookie），豁免 |
| 越权 | 每个 service 入口 @RequirePermission 注解 + 数据范围拦截器 |
| 限流 | 网关 100 req/s/用户；webhook 路径独立 50 req/s |
| 回调安全 | IM 验签 + 时间戳防重放 + 幂等键 |
| 密钥 | 全走环境变量；严禁入库/入仓；轮换手册入 runbook |
| 附件 | MinIO 预签名 URL 上传下载；类型白名单 + 10MB 限 + 病毒扫描钩子（P2） |

## 5. 隐私与合规

- 群消息归档属于企业数据：在需求群公告中明示"本群消息将归档至需求管理平台"
- 个人信息最小化：仅同步姓名/部门/IM ID，不同步手机号邮箱（除非通知需要）
- 解散群前导出完整记录，防止"群散档失"
- 私有化部署：全部数据（含 LLM 调用）可配置走内网国产模型，不出企业边界
