# 07 · 分期路线与验收标准

> 文档版本：v1.0 | 日期：2026-07-17

---

## 0. 前置验证（动工前 1 周）——技术红线排除

| # | 验证项 | 方法 | 通过标准 |
|---|--------|------|---------|
| V1 | 飞书建群/改名/公告/解散 API | 脚本直连 OpenAPI | 全链路无权限报错 |
| V2 | 群内卡片按钮回调 | 自建应用沙箱 | 按钮点击 ≤1s 回调到本地（内网穿透） |
| V3 | 群消息事件订阅 | 同上 | 发消息 ≤2s 收到事件 |
| V4 | 群消息历史导出 | im/v1/messages 分页拉取 | 可全量拉取测试群历史 |
| V5 | 免登 OAuth | H5 页取 code 换 user 信息 | 可拿到 open_id |
| V6 | LLM 澄清对话 | OpenAI 兼容 API 多轮对话 | 可稳定产出结构化字段 |

> V1-V5 任一失败 → 重评 IM 方案（如改企微优先或降级群外审批）；V6 失败 → AI 助手降级为模板引导。

## 1. 阶段总览

| 阶段 | 周期(估) | 目标 | 解决痛点 |
|------|---------|------|---------|
| **P0 MVP** | 6-8 周 | 飞书单平台闭环：建群→群内评审→基线→解散归档 | 说不清/做偏了/查不到（基础版） |

## 2. P0 详设（MVP）

### 2.1 功能清单

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 需求 | 模板化创建/编辑(TipTap)/版本/干系人管理 | Must |
| 状态机 | 全状态流转 + 权限校验 + 审计 | Must |
| 群引擎 | 飞书自动建群/拉人/公告更新/解散 | Must |
| 流程 | 评审卡片(会签) + 基线签认卡片(含指纹) | Must |
| 书记员 | 群消息归档(事件订阅→MQ→PG) | Must |
| 通知 | 待办卡片推送 + 48h 提醒 | Must |
| Web 端 | 需求列表/详情/版本 diff/时间线/归档查看 | Must |
| 组织 | 飞书通讯录同步 + 免登 SSO | Must |
| 搜索 | 需求全文检索(Meilisearch) | Must |
| 基础部署 | Docker Compose + HTTPS + 每日备份 | Must |
| 报表 | 简化版(状态分布/周期) | Should |
| 需求池视图 | 看板(按状态分列) | Should |

### 2.2 明确不做（防范围蔓延）

- ✗ 企业微信、钉钉
- ✗ AI 澄清/RAG（P1/P2）
- ✗ 变更管理（P0 基线后修改=打回重走，先严后松）
- ✗ Git 追溯（P2）
- ✗ 多产品线复杂权限（P0 单产品线即可）

### 2.3 验收标准（可测试）

| # | 场景 | 验收条件 |
|---|------|---------|
| A1 | 创建需求 | 飞书工作台 H5 提交表单 → ≤5s 自动建群且干系人全部入群 |
| A2 | 群内评审 | 评审卡片发出 → 3 人依次点"通过" → 卡片进度实时更新 → 自动进入待签认 |
| A3 | 基线签认 | 业务负责人群内点"确认基线" → 平台出现指纹一致的 baseline 快照 |
| A4 | 越权防护 | 非评审人点按钮 → toast 拒绝 + 审计记录 |
| A5 | 归档 | 需求归档 24h 后群解散 → 平台可看完整群消息时间线 + 搜索命中 |
| A6 | 幂等 | 同一卡片回调重发 3 次 → 仅 1 条投票记录 |
| A7 | 免登 | 飞书工作台点应用 → 无感登录进需求列表 |
| A8 | 状态机铁律 | 直接 UPDATE DB 跳过流转外，任何 API 无法非法流转状态 |
| A9 | 备份 | 删库后可用昨日备份恢复（演练一次） |
| A10 | 性能 | 列表页 P95 < 500ms（1 万需求数据量） |

### 2.4 里程碑

```
W1    技术红线验证(V1-V6) + 骨架工程 + 飞书应用注册
W2-3  需求域 + 状态机 + 模板 + Web 基础页面
W3-4  飞书集成: SSO/建群/卡片/事件订阅
W5    评审+基线闭环 + 书记员归档
W6    解散归档 + 搜索 + 待办 + 部署备份
W7    联调修复 + A1-A10 验收测试
W8    试点业务线真实需求跑通 + 复盘
```

## 3. P1 要点

- AI 澄清助手（群内机器人对话式采集，LLM 抽象层可换模型）
- 变更管理全流程（变更卡片 + diff 通知 + major 重签基线）
- 企微接入（降级模式：群沟通 + 私聊审批卡片）
- 超时升级（72h 抄送上级 / 7d 阻塞标记）
- 保密需求模式
- 基线 PDF 导出存证

## 4. P2 要点

- Git webhook 自动关联（分支/commit 含 REQ 号）+ CI 结果群内播报
- 追溯矩阵视图 + 测试用例关联
- 报表全量：返工率 / 一次通过率 / 澄清周期 / 验收积压
- RAG 语义问答（群归档+文档+审批记录向量化）
- 审计 hash 链防篡改

## 5. 风险登记册（Top 5）

| # | 风险 | 概率 | 影响 | 对策 |
|---|------|------|------|------|
| R1 | 飞书卡片回调在内网不通 | 中 | 高 | 前置 V2 验证；备案：natapp/内网穿透或回调网关放 DMZ |
| R2 | 群爆炸用户反感 | 中 | 中 | 归档即解散 + 待办聚合 + 小需求合并建群策略(P1) |
| R3 | 业务方仍绕开平台口头定 | 高 | 高 | 制度配套："无基线不排期"写入研发流程；AI 打标把群内决议自动拉回平台 |
| R4 | LLM 输出不稳定 | 中 | 低 | AI 仅辅助不阻塞；失败降级纯模板 |
| R5 | P0 范围蔓延 | 高 | 中 | Must 清单冻结；新增需求进 P1 backlog |

## 6. 与制度配套的建议（技术之外）

1. 研发排期会上，PM 出示 QM 中"已基线"需求列表，**不在列表不排期**
2. 业务负责人考核加入"基线签认及时率"（系统可出数据）
3. 返工复盘会必查 QM 变更记录，定位"哪一环没说清"
